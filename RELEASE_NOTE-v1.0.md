# Release Note — v1.0

初版发布，实现 Jazz 乐谱浏览与管理的核心功能闭环。

## 当前规模

- **11566 首曲目**（21 首已软删除）
- **10 个 PDF 分册**：557 Jazz Standards、The New Real Book Vol.1-3、The Real Book Of Blues、The Real Book 6th Edition (Eb)、The Rea Easy Book Vol.1-2、乐队总谱合集、简谱合集

## 功能列表

### 乐谱浏览
- 三栏布局：曲目列表 | PDF 乐谱显示 | 信息编辑
- 搜索框实时过滤（中/英文曲名、别名），支持 `*` 通配符
- 按分册筛选（下拉菜单，自动从 `pdf_repo/` 读取）
- 按页码顺序 / 录入顺序 / 难度升序 / 难度降序 排序
- 仅收藏 / 被删除 筛选
- 列表懒加载：初始 100 首，滚动到底部自动加载更多，万级数据流畅

### PDF 乐谱显示
- PyMuPDF 200 DPI 高清渲染
- 整页适配窗口，无滚动条完整显示
- 自适应窗口缩放（resize 实时重渲染）
- 滚轮缩放（1x ~ 8x），以鼠标位置为中心
- 缩放后可拖拽平移，带边界限制
- 多页歌曲翻页（按钮 + 左右方向键）
- 右键面板 ◀▶ 逐页预览定位

### 信息编辑
- 可编辑字段：曲名、译名、别名、难度（1-9）、起始页码、占用页数、分册（可编辑下拉）、备注
- 收藏 ★ 即时保存 / 软删除 / 恢复 / 彻底删除
- 手动新增曲目（New 按钮 / Ctrl+N），自动分配序号
- 保存后焦点自动回到列表

### 批量操作
- 多选模式（Ctrl/Shift + 点击），选中 ≥2 首自动进入
- 批量删除（仅非删除项）/ 批量恢复（仅已删除项）
- 彻底删除（需二次确认）
- 操作完成后自动退出多选模式并刷新列表

### 数据管理
- SQLite 持久化存储，首次启动自动建库
- 从 Markdown 表格批量导入（增量：仅替换涉及的分册）
- 智能分册名匹配（自动处理大小写、后缀缺失/错误）
- 选中曲目导出为 Markdown 表格（Ctrl+E）
- OCR 结果自动纠偏脚本（`apply_ocr_results.py`，位置约束匹配）

## 技术栈

- **GUI**: PyQt5 (Fusion 风格)
- **PDF 渲染**: PyMuPDF (fitz)
- **数据**: SQLite3
- **语言**: Python 3.10+

## 问题修复

### 2026-06-12
- **搜索筛选时键盘导航跳闪**：输入关键词筛选后，用上下方向键 navigate 歌曲时列表上下跳动，选中项不在可见范围内。根本原因是 `QListWidget.setHidden()` 隐藏不匹配项后，Qt 内部键盘导航仍会遍历隐藏项导致滚动异常。改为筛选时直接重建列表（只放入匹配项），不再使用 setHidden/ItemIsSelectable 方案。

## 已知限制

- 多页乐谱暂不支持并排显示，需翻页浏览
- 部分 OCR 自动导入的曲名需手工修正（尤其是 Eb 版 Real Book 手写体扫描，识别率低）
- 无 PDF 全文搜索（仅按曲名/译名/别名搜索数据库字段）
- 无单元测试覆盖
